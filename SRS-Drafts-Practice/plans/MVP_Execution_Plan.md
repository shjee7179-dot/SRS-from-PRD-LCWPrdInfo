# MVP Vibe Coding 최적화 실행 계획 (Scope-Down Strategy)

본 문서는 `SRS-001_VPS_v1.0_Opus.md` 명세서를 100% AI 바이브코딩(Vibe Coding) 기반으로 안정적이고 빠르게 구현하기 위해, 외부 표준 기술(LTI, xAPI, Open Badge) 연동을 후순위로 미루고 **내부 독립 시스템(Supabase + Next.js)**만으로 Phase 1 로드맵을 구성한 실행 계획서입니다.

---

## 1. 단계별 구현 대상 (Phase Scope)

빠른 MVP 출시와 AI 환각(Hallucination)에 의한 연동 오류를 방지하기 위해 다음과 같이 2단계로 분리합니다.

### Phase 1: 자체 생태계 완결형 (Target)
- 외부 시스템 접속 없이, Next.js 단일 컨텍스트와 내부 DB(Prisma/Supabase)만으로 동작하는 핵심 비즈니스 로직.
- **F1 (AI 업로드)**: 엑셀 파일 수동 업로드 + Vercel AI SDK (Gemini) 정규화
- **F2 (데이터 허브) / F6 (대시보드)**: 정규화된 데이터를 DB에 CRUD하고 차트로 렌더링
- **F4 (배지 발급) / F5 (인재 매칭)**: 시스템 내부 DB 상태값(`is_badge_issued: true`)에 기반한 인재 필터링 및 프로필 UI 노출 (단, 외부 오픈배지 지갑으로의 Export는 미지원)

### Phase 2: 외부 표준 프로토콜 부착 (Future)
- Phase 1이 정상 작동하여 실질적인 운영이 일어나는 시점에 점진적으로 추가.
- LMS 연동 (LTI 1.3), 학습 활동 스트리밍 (xAPI LRS), 오픈배지 공식 스펙 (IMS 2.0 Assertion 발행)

---

## 2. 핵심 사용자 경험 (Value Delivery) 훼손 여부 검토

이러한 Scope-Down(범위 축소)을 적용했을 때, 서비스가 고객들에게 제공해야 할 최우선 가치(Aha Moment)가 훼손되는지 분석합니다.

### 🟢 [고객군 1] 사업관리 매니저 (핵심 가치: 수기 작업 축소)
- **가치 훼손 발생 여부: 없음 (0%)**
- **검토 내용**: 매니저는 "다양한 서식의 기관 엑셀 파일을 업로드만 하면 AI가 알아서 매핑해 준다"는 핵심 효용을 온전히 누릴 수 있습니다. xAPI LRS 전송이 뒤로 미뤄져도, 내부 DB에 통합 데이터가 적재되어 있으므로 무결성 리포트 다운로드(F3) 경험은 완벽히 동작합니다.

### 🟢 [고객군 2] 앵커기업 HR (핵심 가치: 검증된 맞춤형 인재 확보)
- **가치 훼손 발생 여부: 없음 (0%)**
- **검토 내용**: 기업 HR의 궁극적 목적은 "우리 지역(Local Fit)에 맞는 역량 검증 인재 목록을 리스트업하고 제안을 보내는 것"입니다. 지원자가 가진 배지가 국제 표준(Open Badge 2.0) 해시로 증명되었는지 여부보다는, **플랫폼 내부에서 공신력 있는 배지로 렌더링되고 검색 필터에 걸린다는 사실 자체**가 핵심입니다. 따라서 플랫폼 내 채용 매칭 가치는 전혀 훼손되지 않습니다.

### 🟡 [고객군 3] 수료생/학습자 (핵심 가치: 성취감 및 커리어 자산화)
- **가치 훼손 발생 여부: 미미함 (10%)**
- **검토 내용**: 사용자는 플랫폼에 로그인했을 때 자신이 달성한 'Local Fit 배지'를 화려한 UI 애니메이션과 함께 확인할 수 있어 일차적 성취감을 얻습니다. 단독으로 외부 블로그나 링크드인에 "공식 오픈배지"를 복사해서 붙여넣는 기능은 Phase 2로 연기되나, 이 MVP의 핵심 목표(기업-인재 연결)를 고려할 때 Phase 1 가치로는 충분히 타협할 만한 수준입니다.

**✅ [결론]**
외부 표준 프로토콜 부재는 MVP의 '핵심 고객 여정'을 단절시키지 않으며, 오히려 바이브코딩을 통한 **초고속 개발 스피드**로 핵심 가치를 시장에 2배 이상 빨리 검증할 수 있게 하므로 **강력히 권장되는 접근**입니다.

## 3. 미결 사항 (Open Issues)

본 계획서 추진에 앞서 추가 확정이 필요한 파생 이슈 3건입니다.

1. **LLM 활용 범위 한계선 설정**
   - 현재 AI는 엑셀 스키마 매핑(F1)에만 사용 제안 중. 이후 무결성 리포트의 피드백 코멘트 작성 등 추가 확장이 필요한지, 혹은 토큰 비용 절감을 위해 MVP에서는 매핑 전용으로만 가둘지 확정이 필요합니다.
2. **Push / 이메일 노티스 전환 방식**
   - 배치 작업(리포트 생성 등) 완료 시 사용자 알림을 인앱 노티 기능(DB 기록)으로 끝낼 것인지, 실제 외부 이메일 API(Resend, AWS SES 등)를 초기부터 연동할지 결정해야 합니다 (바이브코딩 난이도에 큰 영향을 미침).
3. **Vercel Plan 옵션과 엑셀 처리량 충돌**
   - Vercel Serverless 함수의 무료(Hobby) 계정 한도는 10초, Pro 계정은 5분입니다. 대량의 엑셀 파일을 AI로 파싱할 때 Vercel Timeout이 발생할 리스크가 존재합니다. 
   - 대안: 1) 클라이언트 측에서 엑셀 청크 로딩(Client-side AI SDK), 2) Vercel Background Jobs/Inngest 도입, 3) Vercel Pro 업그레이드 등 대응책 선정이 필요합니다.
